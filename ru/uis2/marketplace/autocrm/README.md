# Интеграция с AutoCRM (CRM Автодилер)

AutoCRM — это специализированная CRM-система «Автодилер» от российской компании INFOTECH, предназначенная для автоматизации автосалонов, автохолдингов и дистрибьюторов автомобильных брендов.

## Функционал интеграции

<details>
<summary style="font-weight:bold;"> Подробнее </summary>
<br />

**Данные передаваемые по звонкам:**

- автоматическое создание клиентов и/или лидов по первичным и повторным обращениям;
- передача информации в лид о UTM-метках звонка (UTM Source, UTM Medium, UTM Campaign, UTM Content, UTM Term);
- звонок по клику из CRM (через расширение UIS);
- создание задач по потерянным звонкам.

**Данные передаваемые по текстовым коммуникациям:**

- встроенное рабочее место оператора (РМО, через расширение UIS) для переписки в онлайн-чате, мессенджерах и соцсетях;
- автоматическое создание клиентов и/или лидов при поступлении чатов и заявок;
- передача информации в лид о UTM-метках обращения (UTM Source, UTM Medium, UTM Campaign, UTM Content, UTM Term);
- создание задач по потерянным чатам.

**Данные получаемые по сделкам:**

- выбор автосалона для выгрузки данных;
- выбор категорий рабочих листов для загрузки в UIS;
- сумма сделок;
- воронки продаж;
- этапы сделок.

</details>

## Подключение учетной записи

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

**Для работы на старой линейке тарифных планов:**

- **Телефония** — Базовые отраслевые решения / API расширенный;
- **Текстовые коммуникации** — Базовые отраслевые решения / API расширенный + Омни Чаты;
- **Получение сделок** — Базовые отраслевые решения / API расширенный + CU_Сквозная Аналитика.

**Для работы на новой линейке тарифных планов:**

- **Телефония** — Инструменты интеграции;
- **Текстовые коммуникации** — Инструменты интеграции + Омни Чаты;
- **Получение сделок** — Инструменты интеграции + Аналитика продаж.

1. В ЛК UIS откройте раздел **Интеграции → Все интеграции**, найдите **AutoCRM (CRM Автодилер)** и кликните на нее, чтобы подключить интеграцию.
2. Нажмите **"Добавить учетные данные"** и заполните поля:
   - **Название** — простое, понятное вам;
   - **AutoCRM URL** — адрес вашей CRM в формате `https://YOURDOMAIN.autocrm.ru`;
   - **Bearer token** — токен для корректной работы интеграции; для его получения необходимо самостоятельно обратиться в техническую поддержку AutoCRM.
3. Чтобы убедиться, что все настроено корректно, кликните на **"Проверить соединение с аккаунтом CRM"**. В случае успеха вы увидите уведомление о том, что синхронизация прошла успешно. Если будет отображаться ошибка — убедитесь, что авторизационные данные (AutoCRM URL и Bearer token) введены корректно.

</details>

# 1. Телефония

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Для передачи данных по звонкам из UIS в AutoCRM (CRM Автодилер) прожмите свитч активации интеграции **"Телефония"**.

При включении интеграции доступны следующие разделы настроек: Основное и Шаблоны.

<br />

## Основное

<br />

1. **Фильтровать звонки** — данная настройка необходима для фильтрации звонков по определенным условиям. При включении настройки станут доступны поднастройки: **"Фильтр входящих"** и **"Фильтр исходящих"** звонков. Доступен одинаковый набор параметров для фильтрации: виртуальный номер, название сценария, сайт, источник, рекламная кампания. Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой в верхнем правом углу.

   **Важно!** Если вы включили чекбокс «Фильтровать звонки», но не выбрали никакую настройку в выпадающем списке — в CRM будут передаваться все звонки.

2. **Входящие звонки** — гибкая настройка создания сущностей в CRM по первичным и повторным обращениям:
   - **Первичные обращения** — выберите, какие сущности создавать (клиент / лид / клиент и лид / ничего).
   - При создании **лида** укажите автосалон, тип обращения, цель обращения и источник обращения.
   - При создании **клиента** укажите тип клиента (физическое или юридическое лицо).
   - **Повторные обращения** — настройте, нужно ли всегда создавать лид или ничего не создавать. При выборе создания лида отобразятся дополнительные настройки (автосалон, тип, цель и источник обращения).

3. **Исходящие звонки** — настройки для исходящих звонков полностью аналогичны настройкам для входящих:
   - **Первичные обращения** — выберите, какие сущности создавать (клиент / лид / клиент и лид / ничего).
   - При создании **лида** укажите автосалон, тип обращения, цель обращения и источник обращения.
   - При создании **клиента** укажите тип клиента (физическое или юридическое лицо).
   - **Повторные обращения** — настройте, нужно ли всегда создавать лид или ничего не создавать. При выборе создания лида отобразятся дополнительные настройки (автосалон, тип, цель и источник обращения).

4. **Задачи** — если звонок будет пропущен, в AutoCRM (CRM Автодилер) автоматически создастся задача:
   - выберите, для каких звонков создавать задачу (входящие / исходящие / входящие и исходящие / ничего не создавать);
   - укажите автора и исполнителя задачи;
   - укажите длительность задачи в часах (ко времени завершения звонка прибавится данное время).
   - Название задачи задается в разделе **"Шаблоны"**.

<br />

## Шаблоны

<br />

В этом разделе вы можете настроить шаблон заголовков клиентов, лидов и задач, которые будут автоматически создаваться в AutoCRM (CRM Автодилер) при поступлении входящих или исходящих звонков.

Если вы не создадите свои пользовательские шаблоны, то будет использоваться универсальный шаблон для каждой не заданной комбинации параметров (шаблон по умолчанию):
- Для лидов: `Новый лид по звонку с +{{contact_phone_number}} из UIS` / `Новый лид по звонку на +{{contact_phone_number}} из UIS`
- Для клиентов: `Новый клиент по звонку с +{{contact_phone_number}} из UIS` / `Новый клиент по звонку на +{{contact_phone_number}} из UIS`
- Для задач: `Новая задача по звонку с +{{contact_phone_number}} из UIS` / `Новая задача по звонку на +{{contact_phone_number}} из UIS`

1. Нажав на кнопку **"плюса"**, можно выбрать переменную для вставки в шаблон названия (номер телефона абонента, виртуальный номер, название сценария и другие данные).

2. Кнопка с иконкой **(глаза)** позволяет сделать предпросмотр наименования, которое будет отображено в создаваемой сущности.

3. Кнопка с иконкой **(закругленной стрелочки)** возвращает шаблон поля к значению по умолчанию.

# 2. Текстовые коммуникации

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Для передачи данных по чатам и заявкам из UIS в AutoCRM (CRM Автодилер) прожмите свитч активации интеграции **"Текстовые коммуникации"**.

При включении интеграции доступны следующие разделы настроек: Основное и Шаблоны.

<br />

## Основное

<br />

### Заявки

1. **Первичные обращения** — выберите, какие сущности создавать (клиент / лид / клиент и лид / ничего).
2. При создании **лида** укажите автосалон, тип обращения, цель обращения и источник обращения.
3. При создании **клиента** укажите тип клиента (физическое или юридическое лицо).
4. **Повторные обращения** — настройте, нужно ли всегда создавать лид или ничего не создавать. При выборе создания лида отобразятся дополнительные настройки (автосалон, тип, цель и источник обращения).
5. **Фильтры** — настройка для фильтрации заявок по определенным условиям (название сайта, рекламная кампания, источник, тип заявки). Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой в правом верхнем углу. Если фильтр не настроен, будут передаваться все заявки.

### Чаты

1. **Первичные обращения** — выберите, какие сущности создавать (клиент / лид / клиент и лид / ничего).
2. При создании **лида** укажите автосалон, тип обращения, цель обращения и источник обращения.
3. При создании **клиента** укажите тип клиента (физическое или юридическое лицо).
4. **Повторные обращения** — настройте, нужно ли всегда создавать лид или ничего не создавать. При выборе создания лида отобразятся дополнительные настройки (автосалон, тип, цель и источник обращения).
5. **Создавать клиента / лид** — выберите событие, при котором необходимо создавать выбранные сущности (на начало чата / на завершение чата).

   **Важно!** При поступлении чата без контактных данных (email или телефон) будет создан только лид.

6. **Фильтры** — настройка для фильтрации чатов по определенным условиям (название сайта, рекламная кампания, источник, канал, тег). Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой в верхнем правом углу. Если фильтр не настроен, будут передаваться все чаты.

### Задачи

1. Включите чекбокс **"Создавать задачу на пропущенные чаты"**.
2. Укажите автора и исполнителя задачи.
3. Укажите продолжительность задачи в часах (ко времени завершения чата прибавится данное время).
4. Название задачи задается в разделе **"Шаблоны"**.

<br />

## Шаблоны

<br />

В этом разделе вы можете настроить шаблон заголовков клиентов, лидов и задач, которые будут автоматически создаваться в AutoCRM (CRM Автодилер) при поступлении заявок и чатов.

Если вы не создадите свои пользовательские шаблоны, то будет использоваться универсальный шаблон для каждой не заданной комбинации параметров (шаблон по умолчанию):
- Для лидов: `Новый лид по заявке {{communication_id}} из UIS` / `Новый лид по чату {{visitor_id}} из канала {{chat_channel_name}}`
- Для клиентов: `Новый клиент {{visitor_phone_number}} по заявке из UIS` / `Новый клиент по чату {{visitor_id}} из канала {{chat_channel_name}}`
- Для задач: `Новая задача по заявке {{visitor_phone_number}} из UIS` / `Задача по чату с посетителем {{visitor_id}} из канала {{chat_channel_name}}`

1. Нажав на кнопку **"плюса"**, можно выбрать переменную для вставки в шаблон (сайт, телефон посетителя, тип заявки, источник обращения и другие данные).

2. Кнопка с иконкой **(глаза)** позволяет сделать предпросмотр наименования.

3. Кнопка с иконкой **(закругленной стрелочки)** возвращает шаблон к значению по умолчанию.

# 3. Передача сделок

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Этот блок предназначен для передачи данных о сделках (рабочих листах) из AutoCRM (CRM Автодилер) в UIS.

1. **Выберите автосалон** — выберите автосалон из списка, по которому будет выполняться выгрузка данных.
2. **Выберите категории рабочих листов** — выберите категории РЛ (аналог сущности «сделка» в UIS), которые нужно загружать в UIS. Доступны категории:
   - трейд-ин;
   - продажа Б\У;
   - сервис;
   - отдел продаж;
   - кузовной ремонт.
3. Нажмите **"Сохранить"**.

**Важно!** Данные по сделкам забираются раз в сутки, в 00:00. После подключения интеграции рабочие листы из AutoCRM (CRM Автодилер) будут попадать в раздел **Отчеты → Сделки** в UIS. Там можно будет найти информацию о категории рабочего листа (данная информация будет отображаться в столбце «Воронка продаж»), этапы сделок, суммы сделок (выручка), контактные данные и так далее.

# 4. Как пользоваться

<br />

<details>
<summary style="font-weight:bold;"> Работа с фреймом РМО </summary>
<br />

**Важно!** Мы гарантируем стабильную работу РМО в Google Chrome, если у вас актуальная версия **[расширения](https://chromewebstore.google.com/detail/uis/nfaolaoglkcldagbicdoppghadihjffl?hl=ru)** и браузера. В других браузерах мы не можем быть уверены в стабильной работе, но стараемся сделать расширение совместимым с другими, похожими на Chrome, браузерами: Яндекс Браузер, Opera, Edge и т. д.

**Установка расширения:**

1. Переходим по ссылке и устанавливаем расширение в магазине Chrome.
2. Нажимаем кнопку **"Войти"** и логинимся под сотрудником UIS, который будет принимать чаты и звонки.
3. После успешного входа в расширение открывается меню: кнопка **"Показать софтфон"** (открывает окно софтфона) и **"Показать чаты"** (открывает окно чатов).

### Чаты

Вызвать окно с чатами можно несколькими способами:

- кликнув на кнопку **Чаты UIS** в боковой панели AutoCRM (CRM Автодилер), вызовется общее окно РМО, в котором можно посмотреть все доступные чаты, а также написать новому клиенту;
- включив свитч в расширении UIS.

Для того, чтобы написать в чат (через подключенный канал чата в UIS), необходимо кликнуть на иконку канала напротив номера телефона (доступно для сущностей лид и клиент).

### Звонки

Софтфон можно отображать всегда:

- активировав свитч **"Показать софтфон"** в настройках расширения;
- кликнув на иконку в боковом меню.

Также софтфон автоматически покажется:

- при входящем звонке;
- при звонке по клику.

Для того, чтобы осуществить звонок по клику (click-to-call), необходимо нажать на номер телефона клиента (доступно для сущностей лид и клиент).

**Преимущества использования расширения UIS:**

- отложенные сообщения;
- расширенная карточка контакта с возможностью тегирования чатов;
- отправка голосовых сообщений;
- просмотр аватарок клиентов при общении с ними в чатах.
