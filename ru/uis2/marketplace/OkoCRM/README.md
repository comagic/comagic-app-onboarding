# Интеграция с OkoCRM

Решение позволяет передавать в наш кабинет данные по сделкам для дальнейшего построения Сквозной аналитики, а также интегрировать функционал телефонии и текстовых коммуникаций, передавая данные по звонкам, чатам и заявкам в OkoCRM.

## Функционал интеграции

<details>
<summary style="font-weight:bold;"> Подробнее </summary>
<br />

**Данные передаваемые по звонкам:**

- синхронизация сотрудников из CRM;
- автоматическое создание контактов и/или сделок по первичным и повторным обращениям;
- передача информации в контакт и сделку о UTM-метках звонка (UTM Source, UTM Medium, UTM Campaign, UTM Content, UTM Term);
- переадресация на персонального менеджера;
- звонок по клику из CRM (через расширение UIS);
- создание задач по потерянным звонкам;
- гибкие настройки передачи звонков в воронки (актуально для сделок).

**Данные передаваемые по текстовым коммуникациям:**

- встроенное рабочее место оператора (РМО, через расширение UIS) для переписки в онлайн-чате, мессенджерах и соцсетях;
- автоматическое создание контактов и/или сделок при поступлении чатов и заявок;
- передача информации в контакт и сделку о UTM-метках (UTM Source, UTM Medium, UTM Campaign, UTM Content, UTM Term);
- создание задач по потерянным чатам.

**Данные получаемые по сделкам:**

- выбор воронок продаж, из которых подтягиваются сделки;
- сумма сделок.

## Подключение учетной записи

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

1. В ЛК UIS откройте раздел **Интеграции → CRM**, найдите **OkoCRM** и кликните на нее, чтобы подключить интеграцию.
2. Нажмите **"Добавить учетные данные"** и заполните поля:
   - **Название** — простое, понятное вам;
   - **API Token** — токен пользователя из OkoCRM. Токен в CRM отображается здесь: **Профиль → Выбираем нужный профиль → листаем ниже чекбоксов**. Все запросы к API будут иметь те же права, что имеет пользователь этого токена.
3. Чтобы убедиться, что все настроено корректно, кликните на **"Проверить соединение с аккаунтом CRM"**. В случае успеха вы увидите уведомление о том, что синхронизация прошла успешно. Если будет отображаться ошибка — убедитесь, что авторизационные данные (ключ API) введены корректно.

## Синхронизация сотрудников

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Синхронизация — это автоматический импорт сотрудников из OkoCRM в UIS. Сопоставление происходит по e-mail, который указан в карточке сотрудника: e-mail в OkoCRM должен совпадать с e-mail в UIS.

1. Выберите отдельно тех, кто будет работать с обращениями (доступен поиск по ФИО), и нажмите **"Синхронизировать сотрудников"**, либо воспользуйтесь чекбоксом **"Синхронизировать всех сотрудников"**.
2. В этом разделе отображается статус, дата и время последней синхронизации сотрудников.
3. Нажмите **Сохранить**.

**Важно!** Если в OkoCRM появился новый сотрудник, которого надо синхронизировать в UIS, то это необходимо сделать вручную по нажатию кнопки **"Синхронизировать"**, после чего сотрудник станет доступен для выбора в настройке, либо дождаться автоматической синхронизации (раз в сутки в 00:00:00).

</details>

# 1. Телефония

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Для передачи данных по звонкам из UIS в OkoCRM прожмите свитч активации интеграции **"Телефония"**.

При включении интеграции доступны следующие разделы настроек: Основное, Шаблоны, Ответственные и Мультиворонки.

<br />

## Основное

<br />

1. **Фильтровать звонки** — данная настройка необходима для фильтрации звонков по определенным условиям. При включении настройки станут доступны поднастройки: **"Фильтр входящих"** и **"Фильтр исходящих"** звонков. Доступен одинаковый набор параметров для фильтрации: виртуальный номер, сценарии, сайты, источники, рекламные кампании. Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой в верхнем правом углу. Если клиент нажал на чекбокс «Фильтровать звонки», но не выбрал никакую настройку в выпадающем списке — передаются все звонки.

2. **Входящие звонки** — гибкая настройка создания сущностей в CRM по первичным и повторным обращениям:
   - **Первичные обращения** — выберите, какие сущности создавать при первичном обращении (контакт / сделка / контакт и сделка / ничего).
   - **Ответственный по первичным обращениям** — выберите, на кого назначать созданные сущности (первый разговаривающий / последний разговаривающий / сотрудник из настроек интеграции).
   - **Повторные обращения** — укажите, нужно ли всегда создавать сделку или нет (создавать сделку всегда / ничего не создавать).
   - **Ответственный по повторным обращениям** — выберите, на кого назначать созданные сущности (ответственный из карточки контакта / последний разговаривающий / сотрудник из настроек интеграции).
   - **Включить переадресацию на персонального менеджера** — активируйте, если необходима переадресация на персонального менеджера из CRM.

   **Важно!** Если в звонке участвовали несинхронизированные сотрудники, ответственного за такой звонок проставляем сотрудника из настройки «Ответственный по умолчанию» (вкладка «Ответственные»).

3. **Исходящие звонки** — гибкие настройки создания сущностей в CRM по первичным и повторным обращениям (аналогичные настройкам по входящим звонкам).

4. **Задачи** — настройка создания задач по потерянным звонкам:
   - **При потерянном входящем звонке создавать задачу** — выберите, на кого создавать задачу.
   - **При неуспешном исходящем звонке создавать задачу** — выберите, на кого создавать задачу.
   - **Тип задачи** — выберите тип создаваемой задачи.
   - **Срок задачи (дней)** — укажите срок исполнения задачи в днях (к времени завершения звонка прибавится данное время).

<br />

## Шаблоны

<br />

В этом разделе вы можете настроить шаблон заголовков контактов, сделок и задач, которые будут автоматически создаваться в OkoCRM при поступлении входящих или исходящих звонков.

Если вы не создадите свои пользовательские шаблоны, то будет использоваться универсальный шаблон для каждой не заданной комбинации параметров (шаблон по умолчанию):
- Для сделок: `Новая сделка по звонку с +{{contact_phone_number}} из UIS` / `Новая сделка по звонку на +{{contact_phone_number}} из UIS`
- Для контактов: `Новый контакт по звонку с +{{contact_phone_number}} из UIS` / `Новый контакт по звонку на +{{contact_phone_number}} из UIS`
- Для задач: `Новая задача по звонку с +{{contact_phone_number}} из UIS` / `Новая задача по звонку на +{{contact_phone_number}} из UIS`

1. Нажав на кнопку **"плюса"**, можно выбрать переменную для вставки в шаблон названия (номер телефона абонента, виртуальный номер, название сценария и другие данные).

2. Кнопка с иконкой **(глаза)** позволяет сделать предпросмотр наименования, которое будет отображено в создаваемой сущности.

3. Кнопка с иконкой **(закругленной стрелочки)** возвращает шаблон поля к значению по умолчанию.

<br />

## Ответственные

<br />

Настройка позволяет управлять назначением ответственных сотрудников за сущности в зависимости от условий.

1. Задайте **Ответственного по умолчанию за входящие звонки** — обязательное поле (этот сотрудник будет указан ответственным в CRM при выборе настройки "сотрудника из настроек интеграции" по создаваемым сущностям).

2. Задайте **Ответственного по умолчанию за исходящие звонки** — обязательное поле.

3. Для добавления условия назначения ответственного нажмите кнопку **"Добавить"**:
   - Выберите сотрудника из выпадающего списка.
   - Далее выберите условия для определения правил (например, можно настроить отдельного ответственного сотрудника для конкретного сайта или виртуального номера).
   - Нажмите **"Сохранить"**.

<br />

## Мультиворонки

<br />

Данная настройка необходима для того, чтобы указать, в какую воронку и этап должны создаваться сделки по обращениям из UIS.

1. Выберите **Воронку по умолчанию для входящих звонков**.
2. Выберите **Воронку по умолчанию для исходящих звонков**.
3. При необходимости настройте по отдельным условиям (используя фильтры) воронку/этап для входящих звонков.
4. Нажмите **"Сохранить"**.

<br />

# 2. Текстовые коммуникации

<br />

<details>
<summary style="font-weight:bold;"> Шаги по подключению </summary>
<br />

Для передачи данных по чатам и заявкам из UIS в OkoCRM прожмите свитч активации интеграции **"Текстовые коммуникации"**.

При включении интеграции доступны следующие разделы настроек: Основное, Шаблоны, Ответственные и Мультиворонки.

<br />

## Основное

<br />

### Заявки

1. **Первичные обращения** — выберите, какие сущности создавать при первичном обращении (контакт / сделка / контакт и сделка / ничего).
2. **Повторные обращения** — укажите, нужно ли всегда создавать сделку или нет (создавать сделку всегда / ничего не создавать).
3. **Ответственный за обращения** — выберите, на кого назначать созданные сущности (ответственный из карточки контакта / сотрудник из настроек интеграции).
4. **Фильтры** — настройка для фильтрации заявок по определенным условиям (сайт, рекламная кампания, источник, тип заявки). Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой в верхнем правом углу. Если фильтр не настроен, то будут передаваться все заявки.

### Чаты

1. **Первичные обращения** — выберите, какие сущности создавать при первичном обращении (контакт / сделка / контакт и сделка / ничего).
2. **Повторные обращения** — укажите, нужно ли всегда создавать сделку или нет (создавать сделку всегда / ничего не создавать).
3. **Ответственный за обращения** — выберите, на кого назначать созданные сущности (ответственный из карточки контакта / сотрудник из настроек интеграции).
4. **Создавать контакт / сделку** — выберите событие, при котором необходимо создавать выбранные сущности (на начало чата / на завершение чата / при простановке тега). При выборе варианта «При простановке тега» будет доступна дополнительная настройка, в которой необходимо будет указать конкретный тег.
5. **Передавать чаты от клиентов без контактных данных** — включите, если необходимо передавать чаты от посетителей без номера телефона. При каждом новом чате будет создаваться новый контакт и сделка, что может привести к дублям в OkoCRM.
6. **Фильтры** — настройка для фильтрации чатов по определенным условиям (сайт, канал, тег, рекламная кампания, источник). Условия можно прописывать через операторы И/ИЛИ. Если фильтр настроен, он будет помечен зеленой точкой. Если чекбокс «Фильтровать чаты» включен, но фильтр не настроен, то будут передаваться все чаты.

### Задачи

1. **При потерянном чате создавать задачу** — выберите, на кого создавать задачу.
2. **Категория задачи** — выберите категорию создаваемой задачи.
3. **Срок задачи (дней)** — укажите срок исполнения задачи в днях (к времени завершения чата прибавится данное время).

<br />

## Шаблоны

<br />

В этом разделе вы можете настроить шаблон заголовков контактов, сделок и задач, которые будут автоматически создаваться в OkoCRM при поступлении чатов и заявок.

Если вы не создадите свои пользовательские шаблоны, то будет использоваться универсальный шаблон для каждой не заданной комбинации параметров (шаблон по умолчанию):
- Для сделок: `Новая сделка по чату {{visitor_id}} с канала {{chat_channel_name}}` / `Новая заявка с сайта под номером {{communication_id}} из UIS`
- Для контактов: `Новый контакт по чату {{visitor_id}} с канала {{chat_channel_name}}` / `Новый контакт {{visitor_phone_number}} по заявке с сайта из UIS`
- Для задач: `Задача по чату с посетителем {{visitor_id}} из канала {{chat_channel_name}}` / `Новая задача по заявке {{visitor_phone_number}} из UIS`

1. Нажав на кнопку **"плюса"**, можно выбрать переменную для вставки в шаблон (номер телефона, виртуальный номер, название сценария и другие данные).

2. Кнопка с иконкой **(глаза)** позволяет сделать предпросмотр наименования.

3. Кнопка с иконкой **(закругленной стрелочки)** возвращает шаблон к значению по умолчанию.

<br />

## Ответственные

<br />

Настройка позволяет управлять назначением ответственных сотрудников за сущности в зависимости от условий.

1. Задайте **Ответственного по умолчанию за чаты** — обязательное поле.
2. Задайте **Ответственного по умолчанию за заявки** — обязательное поле.
3. Для добавления условия назначения ответственного нажмите кнопку **"Добавить"**:
   - Выберите сотрудника из выпадающего списка.
   - Далее выберите условия для определения правил (например, можно настроить отдельного ответственного сотрудника для конкретного сайта или источника).
   - Нажмите **"Сохранить"**.

<br />

## Мультиворонки

<br />

Данная настройка необходима для того, чтобы указать, в какую воронку и этап должны создаваться сделки по обращениям из UIS.

1. Выберите **Воронку по умолчанию для чатов**.
2. Выберите **Воронку по умолчанию для заявок**.
3. При необходимости настройте по отдельным условиям (используя фильтры) воронки/этапы для чатов и/или заявок.
4. Нажмите **"Сохранить"**.
